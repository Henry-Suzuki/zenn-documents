## 1. 綺麗な庭をいじるだけのエンジニアで終わらない

「新しい技術が使えない」「負債だらけで身動きが取れない」。そう言って、多くのエンジニアがレガシーな現場から逃げ出そうとします。
カオス化した現場を「1→100」に耐えうる形に整理できる人材は、市場で不足していると言えます。

「0→1」で新しいものを作る力は、フレームワークの進化によりコモディティ化しつつあります。一方で、止めてはいけないシステム、絡み合ったドメイン、巨大なデータを抱えたまま外科手術を成功させる力は、AI時代でも代替不可能な超・職人技です。

今回は、不満を価値に変えるための職務経歴書・翻訳術を公開します。

## 2. 「やったこと」ではなく、「利益」を書こう

採用担当者やCTOが知りたいのは、どの言語でリファクタリングしたかではありません。その技術介入によって、事業のどの数字が、どう改善したかです。

技術的なアウトプットを「ビジネスのROI（投資対効果）」に翻訳するだけで、あなたの市場価値は上がります。以下のBefore/Afterを見てください。

### 【実践】職務経歴書のBefore/Afterテンプレート

| 改善カテゴリー | Before（よくある書き方） | After（市場価値が上がる書き方） |
| :--- | :--- | :--- |
| **レガシー刷新** | モノリスなシステムの認証モジュールをリファクタリングした | 月間30時間の「火消し対応」を5時間に削減。浮いたリソースを新規機能開発へ転換し、チームのベロシティを1.5倍に向上させた |
| **基盤改善** | データ量増大に伴い、スケーラブルな分散DBへの移行を実施した | 既存DBの性能限界による「事業成長の停止リスク（機会損失）」を解消。理論上、現行の5倍のトラフィックまで耐えうる拡張性を確保した |
| **デリバリー** | CI/CDパイプラインを整備し、テストの自動化率を向上させた | デプロイ頻度を週1回から日次へ改善。リードタイムを60%短縮し、ビジネス要求に対するプロダクトの検証サイクルを2倍に加速させた |
| **トラブル対応** | 頻発していたシステムアラートの原因を特定し、修正した |　年間1,200万円相当の機会損失を生んでいた特定バグを根本解決。SLAを99.0%から99.9%へ引き上げ、顧客離脱率の低下に貢献した |

## 3. 数字がない？数字の捻り出し方

数字は計測されていることが理想ですが、論理的に導き出すこともできます。

1. **工数を人件費に換算する**:
   週に1回、3時間かかっていた手作業を自動化したのであれば、年間約150時間の削減。エンジニアの時給を、たとえば、5,000円とすれば、年間75万円のコスト削減と言い換えられます。
2. **機会損失を仮定する**:
   もしこの改修をしていなかったら、次のセール時にシステムがダウンしていたと予測できるなら、その時の想定損失売上があなたの成果の大きさです。
3. **ベロシティの差分を見る**:
   以前は1つの新機能に1ヶ月かかっていたが、整理後は2週間で出せるようになったなら、開発スピードを2倍にしたという事実が残ります。

## 4. 最後に：カオスを整理する人間が求められる

多くのエンジニアが綺麗な環境を求めて転職活動をします。しかし、企業が本当に高い報酬を払ってでも求めているのは、今ここにあるカオスを整理し、事業を前に進めてくれる人です。

負債に囲まれている今の環境は、見方を変えれば、自分の職務経歴書を最強にするための素材の宝庫であるとも言えます。
